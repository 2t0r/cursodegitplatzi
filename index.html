Comandos utiles de la terminal

clase 11

cd : Moverse entre carpetas

mkdir : crear carpetas

ls : lista de archivos en una carpeta

touch : Crear un archivo

rm [Nombre-archivo] : Borra el archivo

rm -rf [Nombre-carpeta] : Borra carpeta

clear : limpia la terminal

Creando Repositorios

clase 12

git init : nos crea un repositorio de manera local y lo hará en la carpeta donde estamos posicionados o se le puede pasar [nombre_de_la_carpeta] y creará la carpeta con ese nombre.

Los repositorios se crean como carpetas oculas, para eliminarlos puedes usar el explorardor de archivos o rm-rf [nombre del repo]

git add | rm | status (Agregando, quitando y viendo el estatus de archivos)

clase 13

git add [Nombre archivo] Agraga el archivo al staging

git add -A Agraga todos los archivos pendientes al staging

git status Para ver en que estado se encuentran los archivos

git rm --cached :Nos devolvemos un paso

git rm -f [file] :Elimina el archivo por completo.

git add -n [file] : Que no haga nada

git commit (Confirmando cambios)

clase 14

Git commit -m [“mensaje”] : Confirma cambios. Se recomienda ser descriptivo en el mensaje para tener trazabilidad

Git commit -m [“mensaje”] -amend : agregar nuevos cambios al ultimo commit

git log : Nos muestra la historia de todos los commits que hemos realizados.

git tag (Etiquetando confirmaciones)

clase 15

git tag -a -m"": Nos permite agregar etiquetas a nuestros cambios. para agregar tag a otro commit se usa el SHA1

git tag -l: Nos muestra la lista de etiquetas.

git tag -f [SHA1]: Para renombrar. Se coloca el SHA1

git tag -d : Para borar.

git log (Revisando la historia de nuestro proyecto)

clase 16

git log : Muestra el historial de commit con fecha y autor

git log --oneline : Muestra un resumen de los commit

git log --oneline --graph : Muestra la grafica de commit con sus ramas

git log -# : Muestra el numero especifico de commit que quieres ver

git diff (Revisando los cambios entre versiones)

clase 17

git diff [SHA1V1] [SHA1V2]: Muestra la diferencia entre commit

git diff [tagV1] [tagV2]: tambien se pueden usar los tags

git reset -- soft

clase 18

git reset --soft [SHA1]: Desde que version borramos los commit

Nota: Hay que tener en cuenta que si usas git reset --soft Tienes 10 commits y borras el numero 7, los commits 8, 9 y 10 también se borran, y tendras un nuevo commit que tomara el numero 8.

git reset -- mixed

clase 19

git reset --mixed [SHA1]: Elimina los cambios, también del staging area.

Nota: La diferencia con git reset --soft es que esta deja los cambio en staging

git reset --hard

clase 20

git reset --hard [SHA1]: Elimina los cambios por completo.

git branch (Múltiples variantes del repositorio)

clase 22

git branch [nombre]: crea una rama

git branch -d-/D [nombre] elimina una rama

git branch -m [nombre] [nombre nuevo] Renombrar

git branch -l Lista de ramas

git branch (Múltiples variantes del repositorio)

clase 23

git checkout [nombre/sha1]: Nos permite movernos entre ramas o commits, sirve para debugging

git checkout -b [nombre_rama]: Crea una rama si se cambia a esa rama sin necesidad de usar git branch

git merge (Mezclando ramas y resolviendo conflictos)

clase 25

git merge [nombre_rama]: mezcla la rama actual con la rama deseada

git rebase (Reescribe la historia de tu proyecto)

clase 26

git rebase [nombre_rama]: Hace la mezcla de acuerdo al orden cronologico que se fueron creando

se recomiendo no usar esto cuando se esta trabajando en equipo. Usarlo solo de manera local
